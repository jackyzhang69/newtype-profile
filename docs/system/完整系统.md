- EMPLOYER_FRAUD_RISK 徽章完善

- 255行 refusal_patterns.md

### 4.4 MCP 搜索策略文档 ⭐⭐⭐⭐⭐

- 4阶段搜索最清晰
- 1058行完整文档
- 包含预期结果数量

### 4.5 提交信模板 ⭐⭐⭐⭐

- 190行专业模板
- 宗教工人专项提交信
- 配偶工签专项提交信

## 五、Work App 的劣势

### 5.1 类别深度不均 ⚠️

| 类别             | 深度       | 问题                       |
| ---------------- | ---------- | -------------------------- |
| LMIA             | ⭐⭐⭐⭐   | 完整                       |
| R205(d) 宗教工人 | ⭐⭐⭐⭐⭐ | 最详细                     |
| PGWP             | ⭐⭐       | 仅基础规则，缺资格保护策略 |
| ICT              | ⭐⭐⭐     | 缺案例法支持               |
| Open Spouse      | ⭐⭐       | 应与 spousal app 对齐      |
| PNP              | ⭐⭐⭐     | 省级差异未覆盖             |

### 5.2 案例法支持薄弱 ⚠️

- 只有5个主题描述，缺具体 landmark cases
- 对比 Study 的 Singh, Khazaie, Luk 等明确引用
- `caselaw_query_patterns.json` 远不如 Study 的 17K 详细

### 5.3 风险评分缺乏量化 ⚠️

- 有徽章但无分数
- 对比 Study: `fatal(100), high(30), medium(15), low(5)`
- 无阈值触发逻辑

### 5.4 政策更新追踪缺失 ⚠️

- 缺少类似 Study 的 `imm0276_odn_policy.md`
- LMIA 政策变化未追踪
- IMP 更新未覆盖

### 5.5 动态 Prompt 文件缺失 ⚠️

- SKILL.md 引用 `detective_prompt.md` 等
- references/ 中不存在
- 影响知识注入完整性

### 5.6 中文客户指南薄弱 ⚠️

- 对比 Spousal 的 love story template
- 对比 Study 的中文双语支持
- Work 缺少中文版个人陈述模板

## 六、改进方案

### 6.1 短期改进（高优先级）

#### P1: 完善案例法支持

**目标**：对齐 Study 的案例法覆盖水平

**具体任务**：

1. 扩展 `work-immicore-mcp/caselaw_query_patterns.json`
   - 添加 LMIA 相关 landmark cases
   - 添加雇主欺诈相关判例
   - 添加 ICT 相关判例
2. 升级到 v3.0 API 格式
3. 添加 authority verification 机制

**参考**：`.claude/skills/study-immicore-mcp/references/caselaw_query_patterns.json`

#### P2: 风险评分量化

**目标**：建立与 Study 对齐的量化评分系统

**具体任务**：

1. 修改 `risk_badges.json` 添加分数
   ```json
   {
     "severity_scores": {
       "fatal": 100,
       "high": 30,
       "medium": 15,
       "low": 5
     }
   }
   ```
2. 添加阈值触发逻辑
3. 与 Strategist agent 评分逻辑对齐

#### P3: 补充动态 Prompt 文件

**目标**：完善 knowledge-injection 架构

**具体任务**：

1. 创建 `detective_prompt.md`
2. 创建 `strategist_prompt.md`
3. 创建 `gatekeeper_prompt.md`
4. 创建 `reporter_prompt.md`
5. 添加到 references/ 目录

### 6.2 中期改进（中优先级）

#### P4: 深化薄弱类别

**目标**：各类别达到宗教工人级别的深度

| 类别        | 需添加内容                                     |
| ----------- | ---------------------------------------------- |
| PGWP        | 资格保护策略、Part-time 风险、Program 变更影响 |
| ICT         | 专项案例法、L-1B 对比、薪资合规                |
| Open Spouse | 与 spousal app 对齐、关系真实性关联            |
| PNP         | 省级差异（BC/ON/AB/MB/SK）、联邦阶段风险       |

#### P5: 政策更新追踪机制

**目标**：建立类似 Study 的政策追踪

**具体任务**：

1. 创建 `policy_updates.md` 文件
2. 追踪 LMIA 政策变化
3. 追踪 IMP 更新
4. 添加生效日期和影响分析

#### P6: 中文客户指南

**目标**：对齐 Spousal 的客户支持水平

**具体任务**：

1. 创建工作许可个人陈述模板（中文）
2. 创建面试准备指南（中文）
3. 创建文件清单模板（中文）

### 6.3 长期改进（持续优化）

#### P7: 证据标准体系化

- 借鉴 Spousal 的 A/B/C/D 分级
- 建立工作许可特定的证据权重矩阵
- 添加证据充分性标准

#### P8: 跨技能交叉引用

- 建立技能间的明确链接
- 风险徽章 → 文件要求 自动关联
- 类别规则 → 搜索策略 自动路由

#### P9: 省级差异覆盖

- 各省 LMIA 要求差异
- 省提名类别特定规则
- 地区性雇主验证

## 十一、总结

### 核心发现

1. **Work 的独特优势**：类别覆盖最广（10类）、宗教工人最深、搜索策略最清晰
2. **Work 的关键短板**：案例法薄弱、评分无量化、类别深度不均、缺动态Prompt
3. **最佳实践来源**：Study（案例法+量化）、Spousal（客户指南+证据分级）

### 改进优先级

| 优先级 | 任务         | 影响       |
| ------ | ------------ | ---------- |
| P1     | 案例法支持   | 法律准确性 |
| P2     | 风险评分量化 | 评估一致性 |
| P3     | 动态Prompt   | 架构完整性 |
| P4     | 类别深化     | 覆盖质量   |
| P5     | 政策追踪     | 时效性     |
| P6     | 中文指南     | 客户体验   |

### 目标状态

完成改进后，Work app 应达到：

- 案例法覆盖：与 Study 对齐（A+级）
- 风险评估：量化评分体系
- 客户支持：与 Spousal 对齐（中文完整）
- 类别深度：各类别达到宗教工人水平
