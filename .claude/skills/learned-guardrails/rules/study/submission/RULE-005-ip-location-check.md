# RULE-005: IP Address / Location Check Warning

> **类别**: submission (申请提交)
> **严重性**: high
> **适用类型**: study (尤其是境内申请)
> **创建日期**: 2026-01-26
> **来源**: Immigration Lawyer Feedback - Practical Tip (Critical)

## 规则描述

**境内申请 (Inside Canada)** 必须从**加拿大境内**提交。

IRCC 系统会检测申请提交时的 IP 地址和地理位置。如果申请声称是境内申请，但提交时的 IP 地址显示在加拿大境外，**可能导致自动拒绝或严重审查**。

## 原理

1. **系统自动检测**: IRCC 的在线系统会记录提交时的 IP 地址
2. **位置验证**: 境内申请要求申请人在提交时**实际在加拿大境内**
3. **欺诈检测**: IP 地址不匹配被视为潜在欺诈信号
4. **自动触发审查**: 可能导致额外背景调查或直接拒绝

## 触发条件 (Triggers)

当审计境内申请案例时，应**始终**触发此规则提醒：

```
IF application_type == "inside_canada" THEN
  ALWAYS OUTPUT WARNING: "提交位置检查"
  RECOMMENDATION: "确保从加拿大境内网络提交申请"
END IF
```

## 高风险场景

| 场景 | 风险级别 | 说明 |
|------|----------|------|
| 回国探亲期间提交 | **CRITICAL** | 即使有有效学签，从中国提交境内申请会被检测 |
| 使用国外 VPN | **HIGH** | VPN 可能显示为境外 IP |
| 代理人从境外提交 | **HIGH** | 即使是授权代理，也应从加拿大提交 |
| 边境城市使用美国 WiFi | **MEDIUM** | 靠近边境可能连接美国信号 |

## 正确做法

### ✅ 境内申请正确提交方式

1. **在加拿大境内物理位置提交**
   - 使用加拿大的 WiFi 或移动网络
   - 确保 VPN 已关闭（或使用加拿大服务器）

2. **验证 IP 地址**
   - 提交前访问 whatismyip.com 确认 IP 显示加拿大
   - 确保不是使用任何境外代理服务

3. **如果使用代理人**
   - 代理人也应从加拿大境内提交
   - 或由申请人本人在加拿大亲自点击提交

### ❌ 错误做法

| 错误 | 后果 |
|------|------|
| 回国度假时提交境内申请 | 可能被系统标记或自动拒绝 |
| 使用中国 VPN | IP 显示在中国 |
| 父母在国内帮忙提交 | 无论是否有授权，位置不对 |

## 审计输出模板

**对于所有境内申请案例**，在审计报告中加入：

```markdown
### ⚠️ 关键提醒: 提交位置检查 (IP 地址)

**适用场景**: 境内申请 (IMM 5555 / IMM 5709)

**警告**: IRCC 系统会检测申请提交时的 IP 地址。如果申请声称是"境内申请"，但提交时的 IP 地址显示在加拿大境外，可能导致：
- 申请被自动标记为可疑
- 触发额外背景调查
- 严重情况下直接拒绝

**强烈建议**:
1. ✅ **必须**从加拿大境内网络提交申请
2. ✅ 提交前关闭任何 VPN（或确保 VPN 使用加拿大服务器）
3. ✅ 提交前访问 whatismyip.com 验证 IP 显示加拿大位置
4. ❌ **绝对不要**在回国/出境期间提交境内申请
5. ❌ **不要**让境外的家人/朋友帮忙点击提交

**特别注意**: 即使您持有有效学签和回程机票，如果从加拿大境外提交"境内申请"，仍可能被标记为问题申请。
```

## 客户指导清单

审计报告应包含以下提交前检查清单：

```markdown
### 提交前清单 (境内申请)

- [ ] 我目前**人在加拿大境内**
- [ ] 我将使用**加拿大的网络**提交（学校 WiFi、家庭网络、手机数据等）
- [ ] 我已**关闭 VPN** 或 VPN 设置为加拿大服务器
- [ ] 我已访问 whatismyip.com 确认我的 IP 显示**加拿大位置**
- [ ] 我**不会**让境外的任何人帮我点击最终提交按钮
```
