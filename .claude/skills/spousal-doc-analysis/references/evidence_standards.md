# 配偶担保证据质量标准

---

## 一、证据质量评估框架

### 1.1 证据权重分级

| 级别 | 描述 | 示例 |
|------|------|------|
| **A级 (最高)** | 政府签发、难以伪造、直接证明关系 | 共同子女出生证、共同房产证、联合税单 |
| **B级 (高)** | 官方机构签发、有日期戳、可验证 | 共同银行账户、租赁协议、保险单 |
| **C级 (中)** | 第三方文件、有日期、可追溯 | 雇主信、公用事业账单、医疗记录 |
| **D级 (低)** | 自制文件、难以验证 | 个人照片、聊天截图、亲友信 |

### 1.2 证据充分性标准

```
理想证据组合:
┌─────────────────────────────────────────────────────────┐
│  核心证据 (至少3类)                                     │
│  ├─ 共同居住证明 (A/B级)                                │
│  ├─ 财务捆绑证明 (A/B级)                                │
│  └─ 关系发展证明 (时间线覆盖)                           │
│                                                         │
│  支持证据 (至少2类)                                     │
│  ├─ 社会认可证明                                        │
│  └─ 持续沟通证明                                        │
│                                                         │
│  补充证据 (根据情况)                                    │
│  ├─ 共同旅行证明                                        │
│  ├─ 共同参加活动证明                                    │
│  └─ 未来规划证明                                        │
└─────────────────────────────────────────────────────────┘
```

## 二、各类证据评估标准

### 2.1 共同居住证据

#### 高质量指标
- [ ] 连续12个月以上同一地址
- [ ] 多种类型文件相互印证
- [ ] 文件日期覆盖关系发展全程
- [ ] 官方签发或可独立验证

#### 低质量警告
- ⚠️ 仅有一种类型证据
- ⚠️ 地址不一致或变更频繁
- ⚠️ 文件日期集中在申请前短期
- ⚠️ 缺少政府签发文件

#### 评估清单

| 证据类型 | 最低要求 | 理想状态 |
|----------|---------|---------|
| 租赁/房产 | 双方名字出现 | 共同签署 |
| 公用事业 | 至少1份联合账单 | 12个月账单 |
| 银行账户 | 联合账户存在 | 有交易记录 |
| 政府文件 | 同地址 | 驾照+税单+其他 |

### 2.2 财务捆绑证据

#### 高质量指标
- [ ] 联合账户有实际使用记录
- [ ] 共同债务/贷款
- [ ] 互为保险受益人
- [ ] 共同投资/资产

#### 低质量警告
- ⚠️ 联合账户余额极低或无交易
- ⚠️ 财务支持仅单向
- ⚠️ 汇款记录缺乏合理说明
- ⚠️ 财务独立程度过高

### 2.3 沟通记录证据

#### 高质量指标
- [ ] 涵盖关系发展全程
- [ ] 内容显示亲密关系特征
- [ ] 频率与声称的关系深度匹配
- [ ] 包含日常生活细节讨论

#### 低质量警告
- ⚠️ 内容过于肤浅或客套
- ⚠️ 存在长期空白期（无合理解释）
- ⚠️ 通话记录与聊天记录矛盾
- ⚠️ 主要使用语言与申请表声称不符

#### 评估清单

| 沟通类型 | 关注点 |
|----------|--------|
| 聊天记录 | 内容深度、频率、语言、表情使用 |
| 通话记录 | 频率、时长、与重要日期的关联 |
| 邮件 | 正式程度、话题范围 |
| 视频通话 | 频率（长距离关系尤其重要） |

### 2.4 照片证据

#### 高质量指标
- [ ] 不同时间拍摄（跨越数月/年）
- [ ] 不同地点（多个场所）
- [ ] 包含家人/朋友
- [ ] 重要事件记录（婚礼、节日、旅行）
- [ ] 元数据可验证（如适用）

#### 低质量警告
- ⚠️ 所有照片同一时间/地点
- ⚠️ 仅有正式合影，无日常照片
- ⚠️ 无家人或朋友出现
- ⚠️ 照片说明与内容不符
- ⚠️ 明显的后期编辑痕迹

#### 照片提交标准

```
IMM 5533 要求: 最多20张
建议分布:
- 婚礼/订婚: 3-5张
- 日常生活: 5-8张
- 与家人: 3-5张
- 旅行/活动: 3-5张
- 不同时间段: 确保时间跨度
```

### 2.5 亲友证明信

#### 高质量指标
- [ ] 证人与双方均有互动
- [ ] 包含具体事件/日期描述
- [ ] 证人身份可验证
- [ ] 内容与其他证据一致

#### 低质量警告
- ⚠️ 内容过于笼统
- ⚠️ 明显套用模板
- ⚠️ 证人仅认识一方
- ⚠️ 无法联系证人核实

## 三、证据时间线分析

### 3.1 时间线覆盖要求

```
关系时间线应证据覆盖:
┌────────────────────────────────────────────────────────┐
│  相识期          交往期          同居/结婚期          │
│  ──●──────────────●──────────────●──────────────●──    │
│    │              │              │              │      │
│    沟通开始       确定关系       共同生活       申请   │
│                                                        │
│  每个阶段应有至少一类证据覆盖                         │
└────────────────────────────────────────────────────────┘
```

### 3.2 时间线红旗检测

| 红旗 | 描述 | 严重程度 |
|------|------|---------|
| 空白期 | 超过3个月无任何证据 | 高 |
| 突然加速 | 关系在短期内突然升级 | 高 |
| 时间不匹配 | 文件日期与陈述时间线矛盾 | 致命 |
| 申请前集中 | 大部分证据在申请前3个月内产生 | 中 |

## 四、证据真实性验证

### 4.1 文件真实性检查

| 检查项 | 方法 |
|--------|------|
| 官方文件 | 核对签发机构格式、印章、编号 |
| 银行文件 | 检查银行标识、账号格式、日期连续性 |
| 照片 | 检查元数据、背景一致性、PS痕迹 |
| 聊天记录 | 检查平台特征、时间戳连续性 |

### 4.2 内部一致性检查

- [ ] 各文件中姓名拼写一致
- [ ] 地址信息一致
- [ ] 日期无矛盾
- [ ] 工作/学历信息与其他申请材料一致

### 4.3 外部一致性检查

- [ ] 与 IMM 5532 陈述一致
- [ ] 与面试口头陈述一致（如有）
- [ ] 与社交媒体公开信息一致
- [ ] 与之前提交的申请材料一致

## 五、证据缺口处理

### 5.1 可接受的缺口解释

| 缺口类型 | 可接受解释 |
|----------|-----------|
| 无联合账户 | 文化习惯、一方无收入、刚结婚 |
| 无共同房产 | 租房居住、与父母同住 |
| 沟通记录有空白 | 同居期间、技术问题、记录丢失 |
| 照片较少 | 文化原因、隐私偏好 |

### 5.2 必须提供解释信的情况

- [ ] 无法提供至少2类共同居住证据
- [ ] 无法提供至少2类关系证明
- [ ] 存在长期分居
- [ ] 文件缺失或不完整

### 5.3 解释信质量标准

- [ ] 具体说明原因
- [ ] 提供替代证据
- [ ] 前后逻辑一致
- [ ] 语气真诚合理
