## 1.5 R215 申请通道分类 (Application Pathway Classification)

> **⚠️ 关键法律框架 - 必须首先确定申请通道**
>
> R215 条款决定了学习许可申请的法律性质。错误分类会导致申请被拒或要求补充材料。
> **境内申请人必须在评估任何其他因素之前完成 R215 通道分类。**

### 1.5.1 三种申请通道

| 通道 | 法律依据 | 适用情况 | 风险等级 |
|------|----------|----------|----------|
| **延期 (Extension)** | R215(1)(a) | 同一机构 + 同一/相似项目 | 低 |
| **新申请 (New Application)** | R215 | 不同机构 OR 显著项目变更 | 中-高 |
| **恢复 (Restoration)** | R215(1)(b) | 学签过期 < 90天 + 真实学习意图 | 高 |

### 1.5.2 R215 分类决策树

```
┌─────────────────────────────────────────────────────────────────┐
│                    R215 申请通道分类决策树                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Step 1: 当前身份状态检查                                        │
│  ├── 持有有效学习许可? ──────────────────────> 继续 Step 2     │
│  ├── 学签过期 < 90天? ───────────────────────> 恢复申请 (R215(1)(b))│
│  └── 学签过期 > 90天 或 访客身份? ────────────> 新申请 (R215)    │
│                                                                 │
│  Step 2: 机构分析 (仅限 Step 1 为"有效学签")                     │
│  ├── 同一 DLI? ──────────────────────────────> 继续 Step 3     │
│  └── 不同 DLI? ──────────────────────────────> 新申请 (R215)    │
│                                                                 │
│  Step 3: 项目分析 (仅限 Step 2 为"同一 DLI")                     │
│  ├── 同一项目级别 + 同一/相似专业? ──────────> 延期 (R215(1)(a))│
│  └── 不同项目级别 OR 显著专业变更? ──────────> 新申请 (R215)    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.5.3 各通道详细定义

#### A. 延期申请 (Extension) - R215(1)(a)

**法律要求：**
- 持有有效学习许可
- 在**同一指定学习机构 (DLI)** 继续学习
- 项目级别和领域**相同或相似**

**典型场景：**
- 同一大学本科延长学习期限
- 同一学院同一专业从证书升级到文凭
- 同一项目因学业原因需要额外时间

**风险级别：** 低 - 最简单的申请类型

---

#### B. 新申请 (New Application) - R215

**法律要求：** 以下任一条件即触发新申请：
1. **机构变更**: 从一个 DLI 转到另一个 DLI
2. **项目级别显著变更**: 如 Diploma → Bachelor, Bachelor → Master
3. **专业领域显著变更**: 如 Engineering → Business (即使同一机构)
4. **身份状态变更**: 访客身份申请学签

**典型场景：**
- ✅ Langara College Diploma → UBC Bachelor (机构变更 + 级别变更)
- ✅ UBC Bachelor Arts → UBC Master Computer Science (级别变更 + 领域变更)
- ✅ 访客身份申请学习许可

**风险级别：** 中-高
- 机构变更需要解释转学原因
- 项目变更需要证明学术进阶合理性
- **Singh v Canada (2012 FC 526)**: 学术转换受到严格审查
- **Khazaie v Canada (2023 FC 288)**: 学术倒退被视为不合理

---

#### C. 恢复申请 (Restoration) - R215(1)(b)

**法律要求：**
- 学习许可**已过期**
- 过期时间**不超过 90 天**
- 申请人证明有**真实学习意图**
- 在学签有效期内保持合规

**典型场景：**
- 学签过期后发现，立即申请恢复
- 等待新学签期间原学签过期

**风险级别：** 高
- 需要解释为何未及时申请延期
- 需要证明一直保持学习意图
- 90天后无法恢复，必须离境重新申请

### 1.5.4 CHEN Yuxuan 案例分析 (Mock Case)

| 因素 | 情况 | 分类影响 |
|------|------|----------|
| 当前身份 | Visitor Record (学签已过期) | → 非延期 |
| 原机构 | Langara College | |
| 新机构 | University of British Columbia | → 机构变更 |
| 原项目 | Business Management Diploma | |
| 新项目 | Bachelor of Commerce | → 级别变更 |
| **R215 分类** | **新申请 (New Application)** | **非延期** |

**关键判定理由：**
1. 学签已过期 → 不符合 R215(1)(a) 延期条件
2. 机构从 Langara 变更为 UBC → 即使学签有效也需新申请
3. 项目从 Diploma 升级为 Bachelor → 显著项目变更
4. **Singh 先例适用**: Diploma → Bachelor 属于学术转换，需证明合理性

### 1.5.5 风险评估调整

| 申请通道 | 基础评分调整 | 说明 |
|----------|--------------|------|
| 延期 (Extension) | +5 分 | 最低风险，审批率高 |
| 新申请 (New Application) | -3 分 | 需要额外证明材料 |
| 恢复 (Restoration) | -10 分 | 高风险，需要强有力解释 |

### 1.5.6 相关联邦法院判例

| 案例 | 年份 | 关键原则 | 适用场景 |
|------|------|----------|----------|
| **Singh v Canada** | 2012 FC 526 | 学术转换受到严格审查 | 机构/项目变更 |
| **Khazaie v Canada** | 2023 FC 288 | 学术倒退被视为不合理 | Master → Diploma 等 |

## 1.6 Co-op 工作许可 (R186(w)) - "免费附加" 指南

> **⚠️ 重要提示：很多学生不知道的"隐藏福利"**
>
> 如果项目包含 Co-op 或实习组件，可以在申请学签的**同时**申请 Co-op 工作许可。
> 这是一个"免费附加"选项，不收取额外费用，但必须在学签申请时一并提交。

### 1.6.1 Co-op 工作许可资格

| 条件 | 要求 | 说明 |
|------|------|------|
| 项目包含 Co-op | 项目必须包含强制性实习/工作学期 | LOA 应明确标注 |
| Co-op 为课程一部分 | 实习是学位要求 | 非选修实习 |
| 工作时长限制 | Co-op 不超过项目总时长 50% | 法律要求 |

### 1.6.2 法律依据

**IRPR R186(w):**
> 工作许可可签发给：在指定学习机构注册的全日制学生，其学习项目要求工作经验作为课程的一部分。

### 1.6.3 申请流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    Co-op 工作许可申请流程                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Step 1: 确认项目包含 Co-op                                     │
│  ├── 检查 LOA 是否标注 "Co-op" 或 "Work Placement"              │
│  └── 确认 Co-op 是必修要求                                      │
│                                                                 │
│  Step 2: 准备证明文件                                           │
│  ├── Co-op 证明信（学校出具）                                    │
│  └── 项目大纲显示 Co-op 学期                                     │
│                                                                 │
│  Step 3: 同时申请                                                │
│  ├── 在学签申请中勾选 "Co-op Work Permit"                       │
│  └── 上传 Co-op 证明文件                                         │
│                                                                 │
│  ⚠️ 重要: 不额外收费，但必须与学签同时申请                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.6.4 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| ❌ 学签获批后再申请 Co-op 工签 | 需要单独申请，费用 $155 | 与学签同时申请 |
| ❌ 未提供 Co-op 证明信 | 可能被拒或延迟 | 让学校出具证明 |
| ❌ 项目实际无 Co-op 要求 | 申请被拒 | 确认项目确实包含 Co-op |

### 1.6.5 评估检查点

审计时应检查：
- [ ] 项目是否包含 Co-op 组件？
- [ ] 如果有，是否建议客户同时申请 Co-op 工签？
- [ ] LOA 是否明确标注 Co-op 要求？

## 1.7 PGWP 资格保护 - 全日制学习要求

> **⚠️ 对未来 PGWP 的关键影响**
>
> 学习期间的选择会直接影响毕业后工签 (PGWP) 的资格。
> 必须在审计中提醒客户这些潜在风险。

### 1.7.1 PGWP 全日制要求

| 要求 | 说明 | 例外 |
|------|------|------|
| **全日制学习** | 整个项目期间必须保持全日制 | 最后一个学期可以兼职 |
| **最后学期例外** | 仅剩课程不足以构成全日制时可以兼职 | 需要有充分理由 |
| **休学限制** | 休学超过 150 天可能影响 PGWP | 医疗等特殊情况除外 |

### 1.7.2 Part-time/Gap 的影响

| 情况 | 对 PGWP 的影响 | 建议 |
|------|----------------|------|
| 有一学期 Part-time | 可能丧失 PGWP 资格 | 必须有合理解释 |
| 休学超过 150 天 | 严重影响 PGWP 资格 | 医疗证明或特殊情况 |
| 转学导致的 Gap | 需要解释 | 提供转学时间线 |
| 因 COVID 兼职 | 通常可接受 | 保留证据 |

### 1.7.3 审计检查点

审计时应提醒客户：
- [ ] 是否了解 PGWP 需要全日制学习？
- [ ] 如有 Part-time 或 Gap 历史，是否已准备解释？
- [ ] 是否了解休学超过 150 天的后果？
- [ ] 学习计划是否会提及 PGWP 目标？（需谨慎措辞）

## 二、真实学生测试 (Genuine Student Test - R220)

### 2.1 法律基础

**IRPR 220 条款：**
> 签证官不得向外国人签发学习许可，除非确信该外国人：
> (a) 将在授权逗留期限结束时离开加拿大；
> (b) 是真正的学生，会积极追求其课程或学习项目。

### 2.2 评估维度

```
┌─────────────────────────────────────────────────────────┐
│                   真实学生评估框架                       │
├─────────────────────────────────────────────────────────┤
│  1. 学习计划合理性                                      │
│     - 与过往教育的逻辑连续性                            │
│     - 职业目标的可信度                                  │
│     - 选择加拿大/学校的合理理由                         │
│                                                         │
│  2. 财务能力真实性                                      │
│     - 资金来源合法性                                    │
│     - 担保人关系及支持意愿                              │
│     - 投资回报预期合理性                                │
│                                                         │
│  3. 与原籍国联系 (Ties to Home)                         │
│     - 家庭关系强度                                      │
│     - 财产和资产                                        │
│     - 就业前景                                          │
│                                                         │
│  4. 移民历史                                            │
│     - 之前的签证申请                                    │
│     - 旅行合规记录                                      │
│     - 在其他国家的身份状态                              │
└─────────────────────────────────────────────────────────┘
```

### 2.3 双重意图 (Dual Intent - IRPA 22(2))

**法律规定：**
> IRPA 22(2): 外国人寻求临时居民身份的意图与同时寻求成为永久居民的意图并不矛盾。

**关键要点：**
- 双重意图是被允许的
- 但仍需证明：如果永居申请失败，将离开加拿大
- 表述方式极其重要 (参见 Luk v Canada, 2024 FC 623)

- 理想: 6个月以上
- 最低: 3-4个月
- 大额存入: 必须解释来源

## 四、风险评估框架

### 4.1 致命红旗 (Fatal Flags)

- [ ] 无有效 DLI 录取通知
- [ ] PAL 缺失或过期（需要时）
- [ ] 资金明显不足（< 50% 要求）
- [ ] 发现虚假文件
- [ ] 被发现有虚假陈述历史
- [ ] 不可入境（犯罪/安全/健康）

### 4.2 高风险红旗 (High Risk)

- [ ] 学习计划与背景严重不符（倒读）
- [ ] 年龄 > 35岁申请本科或以下
- [ ] 有多次学签拒签历史
- [ ] 家庭可整体移居（无回国约束）
- [ ] 之前在加拿大有违规记录
- [ ] 语言能力与项目要求严重不匹配
- [ ] 申请敏感专业无合理解释

### 4.3 中等风险红旗 (Medium Risk)

- [ ] 年龄 28-35 岁
- [ ] 有1-2次拒签历史
- [ ] 资金历史 < 3个月
- [ ] 学习计划描述模糊
- [ ] 无工作经验直接申请研究生
- [ ] 职业规划不明确
- [ ] 与原籍国联系证据不足

### 4.4 低风险指标 (Positive Factors)

- [+] 年轻单身（18-25岁）
- [+] 专业匹配且有逻辑进阶
- [+] 强约束力（家庭/财产/工作）
- [+] 良好旅行历史（美、英、申根等）
- [+] 知名大学/排名靠前项目
- [+] 奖学金或资助
- [+] 相关工作经验

## 七、检查流程

1. **资格筛查**: Fatal 条件检查 → 任何不满足即停止
2. **风险评估**: 红旗检查 → 记录所有风险点
3. **材料审核**: 文件完整性 → 标记缺失项
4. **综合评估**: 正面因素 vs 风险因素 → 生成风险评分
5. **建议生成**: 基于评估结果提供策略建议
